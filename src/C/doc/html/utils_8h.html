int32_t extractOTP ( char *  hash)

Extrait un entier d'un haché HMAC pour les générateur basés sur cette fonction. D'après la fonction de troncature de la rfc 4226.

Paramètres
[in]hashle haché duquel il faut extraire l'OTP.
Précondition
hash != NULL.
hash est un haché de HMAC_SHA1. sur 20 octets.
Renvoie
un entier sur 31 bits à partir d'un haché; -1 en cas d'erreur.
char* HMAC_SHA1 ( uint64_t  count,
secret  key,
char *  buffer 
)

Remplit le buffer passé en paramêtre avec le HMAC du secret. D'après l'algorithme donné par la RFC 2104

Paramètres
[in]countle compteur passé à hmac pour générer un OTP.
[in]keyla clé secrète pour générer un OTP.
[out]bufferle buffer qui contiendra le résultat
Précondition
key != NULL
buffer != NULL
buffer fait 160 (20 * 8 * sizeof(char)) bits.
Postcondition
buffer contient la valeur correspondant à SHA1(key ^ opad . SHA1(key ^ ipad . count))
où ^ est le XOR bit à bits (^ en C)
et . est la concaténation.
Renvoie
l'adresse de buffer en cas de succès; NULL sinon.
